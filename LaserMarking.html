Software Development for Laser Marking System

Brief

Laser marking uses a focused laser beam to create a permanent mark on object surfaces. The laser marking is efficient, low in pollution, and applicable to various materials. Since laser marking can create high-precision figures, it is popular in both industrial production and commercial products, such as industrial components, packaging, keyboard, home decor gifts, etc.

The system included laser source, galvo scanner, controller, and application software. My responsibility was to develop the application software.

Case Study: Large Area Synchronous Laser Marking

Situation

The laser marking workspace is limited by its mirror system. However, customers want to make a larger pattern, such as marking patterns on door boards. The solution of enlarging the workspace is embeded the laser marking system on a 3-axis machine. The machine can move the laser marking system around to process.

Goal

The laser marking workspace is limited by the galvo scanner module in 100 mm * 100 mm. The goal is to enlarge the workspace. (constrained by the machine workspace)

Method

Through some investigation, there are two approaches:

Approach#1 Positioning Marking

  • Split the pattern into small regions which are equal to the workspace.
  • Move the laser marking module to each region to engrave the pattern.
â†’ Since the distorsion of lens, the margin of neighbor pattern would easily have gaps.

Approach#2 Synchronous Laser Marking

  • Synchronize the axis motion planning and laser marking path. The marking path can continuous without gaps. It could eliminate the gap between different regions.

Next, My challenge is generating tool paths for both machine and the laser marking system and allowing them to coordinate smoothly.

In the beginning, I tried to visualize the problem. I drew the laser marking paths and expected machine paths, and an idea came into my mind. The problem was similar to polyline simplifications. The laser marking path was the original path; the machine instruction path was the simplified path. To build on the initial idea, I further researched several polyline simplification algorithms and examined them with some criteria. For example, the algorithms could easily control the distance between the original line and the simplified line, so the machine path could ensure it covers the laser system path/workspace. Later, to coordinate both paths, the processing parameter should be allocated to each segment appropriately.

Also, to shorten the development process, I developed simulation software to examine the result. (evaluating every modification on machines would waste a lot of time) In the end, I tested the file on the real machine and checked it could pass the designed test cases.

Result

Success to complete the algorithm and complete the evaluation on the machine. This feature attracted new customers for the company. To accelrate the processing speed, the machine and the laser system could not coordinate properly. It required further refinement.
