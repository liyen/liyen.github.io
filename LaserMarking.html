Syntec Laser Marking

Software Development for Laser Marking System

Brief

Laser marking uses a focused laser beam to create a permanent mark on object surfaces. The laser marking is efficient, low in pollution, and applicable to various materials. Since laser marking can create high-precision figures, it is popular in both industrial production and commercial products, such as industrial components, packaging, keyboard, home decor gifts, etc.

The project goal was to offer a comprehensive laser marking solution for customers. The customers could buy the whole laser marking system in one-stop. The system included a laser source, a galvo scanner (an optical mirror module to control the position of the laser beam), a controller, and application software. My responsibility was to develop the application software integrated into the system.

Introduction

Laser marking software allows users to design patterns, set processing parameters, and run the execution files.

Key Feature 1: Draw

Import Files

Import DXF files and pictures.

Draw Geometry

Draw rectangles, ellipses, polylines, etc.

Create Text

Create single-line font and outline font. The single-line font is for quick processing.

Generate Barcode

Generate 1D barcodes and 2D barcodes, e.g., QR code, data matrix, etc.

Serial Number

Users can customize serial number rules. The number would add incrementally on each execution.

Key Feature 2: Edit

Fill

Fill in the selected shape. The filling lines can set the scanning angle, incremental angle, repeated times, etc.

Curved Text

Bend text around the curve.

Cycloid Line

Turn lines into cycloid lines. It can have a widening effect on the pattern.

Combine Geometry

Combine multiple shapes.

Key Feature 3: Execute

Set Laser Source Type and Processing Parameters

Support multiple laser source types and processing parameters, e.g., laser power, frequency, etc.

Red Light Preview

Emit a red light beam for previewing the position of the markings.

Execution

  1. Stand-along Scenario: Connect to the controller and execute.
  2. Industrial Scenario: Put the execution files into the controller and trigger. When integrating the marking system into production lines, the laser marking system can be triggered by other sensors, such as when the camera captures the object.

Case Study: Large Area Synchronous Laser Marking

Problem Statement

The laser marking workspace is limited by its optical mirror system. However, clients want to make a larger pattern, such as patterns on door boards. The solution for enlarging the workspace is to embed the laser marking system on a 3-axis machine. The machine can move the laser marking system around to process.

Goal

The galvo scanner module limits the laser marking workspace to 100 mm * 100 mm. The goal is to enlarge the workspace. The enlarging workspace would rely on the machine workspace.

Method

Through some investigation, there are two approaches:

Approach#1 Positioning Marking

  • Split the pattern into small regions which are equal to the workspace.
  • Move the laser marking module to each region engraving the pattern.
  • The action is moving to the fixed position and marking, then moving to the next fixed position.

→ Because of the lens distortion, the neighboring patterns' margins would easily have gaps.

Approach#2 Synchronous Laser Marking

  • Synchronize the axis motion planning and laser marking paths.
  • The action is moving and marking simultaneously.

→ Since the patterns would be marked continuously, they would not have neighboring gaps.

Next, the challenge was generating tool paths for both machine and the laser marking system and allowing them to coordinate smoothly.

In the beginning, I tried to visualize the problem. I drew the laser marking paths and expected machine paths, and an idea came to mind. The problem was similar to polyline simplifications. The laser marking path was the original path; the machine instruction path was the simplified path. To build on the initial idea, I further researched several polyline simplification algorithms and examined them with some criteria. For instance, the algorithms could easily control the distance between the original lines and the simplified lines, so the laser marking path could ensure in the workspace of the laser marking module while the 3-axis machine moving. Later, to coordinate both paths, the velocities should be allocated to each segment appropriately.

Lastly, I developed simulation software to examine the result in order to shorten the development process. (evaluating every modification on machines would waste a lot of time) Ultimately, I tested the execution files on the machine and passed the designed test cases.

Result

Success in completing the algorithm and completing the evaluation on the machine. This feature attracted new customers for the company.

Reflection

  • I developed several user interface features during the whole development process, such as zoom in/out, select objects, object browsers, etc. In the initial stage, I defined the specification by my idea. But occasionally, I got feedback that users felt they cannot complete their tasks easily and were unsatisfying. After a while, whenever I had to develop user interfaces, I would draw wireframes or build interactive prototypes for users. Then, I could get feedback at the earlier stage of development. It helped me design practical interfaces efficiently.
  • In the Large Area Synchronous Laser Marking project, planning the machine path required knowledge of motion control. At the beginning of the project, I found I needed more resources and asked for help. The research department spent some time teaching our team about the knowledge of motion planning. With the same knowledge base, it was really helpful to streamline the process of later collaboration.
  • Finding early adopters was critical to product development. Since the laser marking market was highly competitive, the early adopters assisted us in knowing which features were attractive to new clients. They played an important role in the decision-making of the product planning strategy. Our team built critical features based on their opinions to draw potential customers.
